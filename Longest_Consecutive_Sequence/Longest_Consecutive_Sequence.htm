     Problem Notes

 

 

 

 

 

 

 

    • 100 doesn't count as it is not "consecutive".
    • 100 is AFTER 4, but the next consecutive number is 5

 

 

 

 

 

 

    • How do we start at the beginning of a sequence?

     

    • Each sequence has a START VALUE with NO LEFT NEIGHBOR

 

 

 

 

    • Convert to set

 

 

 

 

    • 101 in set?  No, so end that sequence, len =1
    • Is 4 a start value, is there a left neighbor?  No not a start value as 3 exists.
    • etc.

 

     Approach # 1 (Wrong) - Sort the Array

    • CAN'T DO THIS AS THE PROBLEM STATES THE ALGORITHM MUST RUN IN O(N) TIME.

 

 

Time Complexity
    •  O(N * Log N  )

     

    • N is the length of array.
    • Time complexity is due to SORTING.

     

Space Complexity

    •  O( 1 )

     

     

     

Summary
    • Don't use this approach at all due to problem instructions.

     

 

 

     Approach # 1 (Best) - Use a Set to Count Consecutive Sequences

 

 

 

Time Complexity
    •  O(N )

     

    • N is the length of the array and we iterate through it once.

     

Space Complexity

    •  O(N)

     

    • As we use memory to create a set.

     

Summary
    • Create a hash set from the array of nums.

     

    • For each number in the array, test if there is a left neighbor (previous num) so we start at the beginning of a sequence.

     

    • If we are at the beginning of a sequence, count the length by testing if the next consecutive num is in the set, incrementing length each time to keep track of the number of consecutive integers.

     

    • Keep track of the maximum length then return it at the end.

     

 

 

 

 

    • Check if start of a sequence

 

 

    • Test if a consecutive number exists in the set, and increase count if yes.
