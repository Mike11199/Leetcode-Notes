     Approach # 1 (Not Best) - Hashmap with Sorted String as Key

 

 

 

Time Complexity
    •  O(N * K * log K)

     

    • N is the length of each strs
    • K is the max len of a string in strs

     

    • The outer loop has complexity O(N) as we iterate through each string. Then, we sort each string in O(K log K) time.

     

     

Space Complexity

    •  O(N * K)

     

    • We have to store N strings of up to length K
    • This is the total info stored in the answer

     

Summary
    • Create a hashmap using defaultdict(list), where each unique key is the sorted string (tuple), and each value is a list of original strings each appended to it.

     

 

 

 

    • Key = the sorted string is the unique key
    • Value = a list of lists, of the original strings which are anagrams to each other
    • Tuple as lists cannot be keys in python.  Tuples are non-mutable (not-changeable).

     Approach # 2 (Best) - Hashmap with Char Frequency Array as Key (Faster as No Sorting)

 

 

 

Time Complexity
    •  O( N * K )
    • N is the length of each strs
    • K is the max len of a string in strs

     

Space Complexity

    •  O( N * K )
    • N is the average length of each string
    • This is the total info stored in the answer

     

Summary
    • Create a hashmap using defaultdict(list), where each unique key is the array of char counts of each letter of the alphabet, and each value is a list of original strings each appended to it.

     

 

    • Key = count of each letter from a to z (26 counts)
    • Value = a list of list of strings which have identical count frequency arrays

 

 

 

 

LETTERS AFTER "a" ARE HIGHER IN # in Unicode SO THIS WORKS.   a=97,  b=98

 

 

 
    • Edge case = use delimiter to avoid situations where keys are the same for different char counts.
