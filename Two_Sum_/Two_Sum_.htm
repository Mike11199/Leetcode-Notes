     Solution #1 (Worst):  Double For Loop.  Brute Force.  O(n^2)

 

 

 

Time Complexity
    •  O(N^2)

     

    • N-Squared as have to have inner/outer loops.
    • TIME LIMIT EXCEEDED

     

Space Complexity

    •  O(1)

     

    • Constant space.

     

Summary
    • Outer loop to traverse through the list's numbers.

     

    • Inner loop to look at all future numbers in the list.

     

    • If a number from the inner loop (nums ahead of outer loop) is the complement needed to reach the target sum, return the inner i and outer j indexes.

 

    • Self refers to the current instance of the class, and is used to access variables belonging to the class.

     

    • Start by defining a function, twoSum that takes in nums, which is expected to be a list of integers, and target, expected to be an integer.  These are "type hints" or "function annotations".

     

    • -> is return value annotation which is just documenting what the function should return.  Another type of function annotation that does nothing.

     Solution #2 (Best):  One Pass Hash Map with Python Dictionary O(N)

 

 

 

Time Complexity
    •  O(N)

     

    • Linear time.  We traverse the list containing n elements only once.
    • Each lookup in the table costs only O(1) time.

     

Space Complexity

    •  O(n)

     

    • Linear space.  Increases with list size.

     

Summary
    • Create a hash map where:

     

    • KEY:         Number (complement to reach target)
    • VALUE:    Index of that number in original list.

     

    • Iterate through list of numbers once.

     

    • If the complement of the current number needed to reach the target sum exists in the hashmap as a key:

     

    • return the index of the current num.
    • return  the index of the complement, which is the hashmap item's value. 

     

    • Else, add the number to the hashmap and continue traversing.

     

 

 

 

 

    • This uses a hash map to only go through each number once and be very efficient.
