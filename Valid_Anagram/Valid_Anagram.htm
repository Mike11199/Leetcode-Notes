     Notes

 

 

 
    • An anagram means that using the characters of one word, we can create another word, using the exact same counts of characters.

     

    • So the STRING LENGTH and the COUNT OF EACH LETTER will be the same.

 

     Approach #1 (Best):  Count the Occurrences of Each Char with Hashmaps (Dictionaries)

 

 

 

Time Complexity
    •  O(N) or O(S+T)

     

    • Linear, as we iterate through both of the strings.

     

Space Complexity

    •  O(N) or O(S+T)

     

    • Linear, as we have to build hashmaps for both strings.

     

Summary
    • Create a hash map for each string where the key is the letter and value is the count by iterating through a string's chars and adding 1 to the value if that char is a key in the dictionary.  Use .get() to avoid key errors.

     

    •  Then iterate through one hash map using its keys to access values to check that each value for a key is equal to that same value for the key in the other string's hash map. Use .get() to avoid key errors.

     

 

    • If we check the length first, we only have to compare one string to the other, and then not the other way around.

     

    • String S and T

    • Use .get() to avoid a KeyError if the key doesn't exist yet.  The second argument is a default value if it does not exist.  Do NOT use += 1 here.
