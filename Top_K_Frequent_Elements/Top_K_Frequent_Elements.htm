Top K Frequent Elements

Wednesday, August 17, 2022

1:35 PM

     Approach # 1 (Best) - Bucket Sort

 

 

 

Time Complexity
    •  O(N) Linear Time

     

    • N is the length of the num array
    • K is the number of elements with the most count to add to result

     

Space Complexity

    •  O(N) Linear Space

     

    • As need to create a dictionary or hashmap.

     

Summary
    • Create a hashmap where Key = num and Value = count.

     

    • Create a frequency array of empty arrays up to len of N+1, as 0 indexed

     

    • Loop through dictionary with enumerate() and append Nums to index of frequency array where index = count.

     

    • Loop backwards through the frequency array to append the highest frequency numbers to the result, with (-1, 0, -1).

     

 

 

 

 
    • MID:  We could have the numbers as keys and count as values, BUT the input values are not bounded.. one could be 1,000,000.  So this would not be linear O(N) time.

 

 

 

 
    • BEST:  Count should be the key as we know the count can't be longer than the list, so that is bounded!
